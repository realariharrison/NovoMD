# Sample descriptor dataset

Precomputed molecular descriptors for 52 well-known compounds, generated by the
open [NovoMD](../README.md) calculator. A small, reproducible give-back: a ready
table you can load without running anything, and a script that regenerates it.

## Files

| File | Description |
|------|-------------|
| `sample_descriptors.csv` | 52 compounds, 28 descriptors each, plus name, category, and SMILES |
| `generate_sample_descriptors.py` | Reproduces the CSV from the public compound list |

## What is in it

Each row is one molecule. The first columns are `name`, `category`, and
`smiles`; the rest are outcome-level descriptors computed from an embedded 3D
conformer:

- **Identity**: molecular weight, atom and bond counts, heavy-atom count
- **Geometry**: radius of gyration, asphericity, eccentricity, inertia shape factor, span, principal moments of inertia
- **Energy** (estimates): conformer energy, van der Waals, electrostatic, torsion strain, angle strain, optimization delta
- **Electrostatics**: dipole moment, total charge, partial-charge range and span, electrostatic potential
- **Surface and volume**: SASA, molecular volume, globularity, surface-to-volume ratio

Per-atom coordinate and bond lists are omitted from the flat table; they are
available from the library (`calculate_properties`).

## Provenance and scope

- **Inputs are public.** The SMILES are textbook structures of common drugs,
  amino acids, simple organics, vitamins, and neurotransmitters.
- **Outputs are open-tool results.** The descriptors are exactly what
  `pip install novomd` produces, computed with RDKit. There are no proprietary
  inputs, models, or enriched fields.
- **Deterministic.** Conformer embedding uses a fixed seed, so the CSV
  regenerates identically.
- **Energy values are estimates** from the conformer, not from a force-field
  simulation. Use them for relative comparison, not absolute prediction.
